Paragraph

The ChocAn System has several requirements that must be met in order for it to operate properly.

What sections will be computerized and how?
  Information such as the data corresponding to services, members, providers, as well as reports
  should be computerized. This should be done by entering the information into a database.
  For the purpose of this project, we will not use an actual database as it is too expensive, but
  we can model/mimic the organization, storage, and function of a database instead.

Where will the data be stored?
  Databases store data in a folder/table format. This ensures easy access and convenience in
  sorting and searching for specific information. This will also protect members' sensitive
  information. We will use a folder/table format to mimic a database.

What hardware will be needed?
  The requirements for a database are: an operating system, a database server, processor cores,
  8+ GB of RAM, and disk space (usually up to/around a few GB). All of the requirements scale up
  in amount/size needed as the data increases. Since we are not using an actual database in the
  project, this can be modeled on our members' laptops/computers (we will not have an actual
  database server).

How will the ChocAn system you develop interact with other systems?
  The ChocAn system must be able to interact with payment systems (EFT and ACME) in order to
  conveniently store both member and provider information, fees, and other data. It should be able
  to store data from the terminal as well. It must also be able to process and store information
  from the communications software (which is being built by another company).

How will users interact with the system?
  Users should be able to access data relating to themselves. They should be able to manage/update
  their personal information. All users should be able to access the list of all services and
  service information as well. It is necessary to have all users perform an authentication process
  to keep data secure. (One example of this is how members must have their card validated by the
  terminal.)
    a. Members should be able to see their updated balance after paying for services
    and/or paying member fees. They will also be able to access and use the communications
    software. They should be able to manage/update their personal information.
    b. Providers should be able to see data relating to themselves, their services, and
    their clients.
    c. Managers should have the most access of all users and should be able to see most
    member/provider info as well as services performed.
